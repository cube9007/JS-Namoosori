실습 전 기초 지식 

mjs 확장자 
script 태그에 타입을 모듈로 주면 모듈로 사용할 수 있음
type="module" src="index.mjs"

규모가 커져서 js코드를 여러 파일에 여러 폴더로 나누어 저장하다보니, 서로가 서로를 효율적으로 불러올 수 있는 시스템이 필요하게 됨


일반적인 js 확장자와의 차이점 
1. import 혹은 export 구문을 사용할 수 있음
2. 별다른 처리를 해주지 않아도 엄격 모드(strict mode)로 동작합니다.
3. 모듈의 가장 바깥쪽에서 선언된 이름은 전역 스코프가 아니라 모듈 스코프에서 선언됩니다.

    *엄격 모드(strict mode)
    느슨한 모드(sloppy mode)의 반대
    strict 모드는 자바스크립트 코드에 더욱 엄격한 오류 검사를 적용해 줍니다.

    선언되지 않은 변수를 사용하면 오류로 인지
    (엄격 모드가 아닐 때는, 자동으로 전역변수로 인지함)
    그 외에 여러 특징들이 있음

export 하는 법 

1. 해당 함수 앞에 export를 붙임 
    export function isNumber( number ) {};

    import할 땐
    import { isNumber } from './~.mjs';

    호출할 땐
    isNumber(10);

2. export default 
    전체에서 하나만 export 함 

    export default {
        isNumber,
        isOperation
    }

    import할 땐
    import validator from './validator.mjs';

    호출할 땐
    validator.isNumber(10);
